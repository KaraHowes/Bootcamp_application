UNDERSTANDING HTTPS

To really define the meaning of HTTPS, we have to go back to good old HTTP. Hypertext Transfer Protocol (HTTP) is a protocol by which files (be it HTML, JavaScript etc) are sent in a network. For a simplified explanation of HTTP, clicky here.

So, what happens when we add an S?.....

In the simplest sense, the S in HTTPS stands for Secure. It is an extension of HTTP which has been encrypted by Transport Layer Security (TLS), which is nicely illustrated in the above Schematic. It's main purpose is to ensure the privacy and integrity of the data across the network. HTTPS sucessfuly prevents Man-in-the-Middle cyberattacks, as well as protecting against eavesdropping, by establishing a secure connection over an insecure network. TLS offers three-layers of protection:

  1. Encryption - The date is encryped to secure it from third-party listeners
  2. Data Integrity - Data can not be interferred with or corrupted
  3. Authentication - proves that the user is communicating with the desired website.

Orignally, HTTPS was an expensive protocol to use, so was only utilised on secured payment transaction services. Now, however, the protocol is more prevalent and is used to maintain website authenticity and to ensure web-browsing remains private. From a web-developers prospective, it is essential to obtain a security certificate, which verifies the website belongs to an organisation.
